# SES新人研修（Javaバックエンド・AI協働）— はじめに必ず読むこと

このフォルダは、**IT未経験で入社した新人**が、3ヶ月（約13週間）で、**AIを使える現場に配属されても即戦力として動ける**ようになるための研修一式です。

> 用語メモ: **SES（エスイーエス）** … お客様の会社（客先）に常駐して、その会社のシステム開発を手伝う働き方のこと。「常駐（じょうちゅう）」とは、自社ではなくお客様のオフィスに出向いて働くことです。

> **この研修が前提とする現場**: あなたは **AIエージェント（Claude / Copilot / Cursor など）を業務で使ってよい現場** に配属されます。だからこの研修は「AIを使わずに全部手で書ける人」を作るのではなく、**AIを正しく使いこなして既存システムに参画し、安全に直して、レビューを通せる人** を作ることを目指します。

---

## 0. この研修のゴール（何ができるようになるか）

3ヶ月後の最終到達点は、ひとことで言うとこれです。

> **「AIを相棒に、既存のシステム（自分が書いたのではない、大きめのコードベース）に参画し、仕様を読み、影響範囲を見極めて安全に改修・小機能追加を行い、テストを書き、PRを出し、レビューの往復をこなして取り込まれるまでを、一人で回せる」**

そのために、研修は **2つのフェーズ** でできています。

### 第1フェーズ（週0〜2）: 基礎を「自分の手で」固める
- **詳細設計（しょうさいせっけい）を読んで、その通りにJavaで実装できる**
  - 「詳細設計」とは、プログラムが何をどう動くべきかを文章や表で細かく書いた指示書のことです。現場では、これを「設計する側」ではなく「読んで作る側」を最初に任されます。
- **テーブルとほしいデータを示されたら、SQLを書いてJavaから実行できる**

> このフェーズではAIにコードを書かせません。**自分で書ける力が、後でAIを使いこなす土台**になるからです。「言語とSQLの素振り」が終わる週2まで、ここは手書きで固めます。

### 第2フェーズ（週3〜12）: AIを「正しく」使って現場に立つ
- **詳細設計を渡されたら、REST APIを1本実装できる**（週3〜・ここからAI解禁）
- **自分が書いたコードに単体テスト（たんたいテスト）を書ける**（週4〜）
- **AIにコードを書かせ、その間違いを見つけて直し、自分の言葉で説明できる**
- **初見の中規模リポを、AIの助けも借りて自力で把握できる**
- **既存リポを壊さずに改修でき、影響範囲を説明でき、全テストをグリーンに保てる**
- **AIにテストを書かせ・レビューさせつつ、最終的な品質判断を自分で持てる**
- **既存システムへの参画（実装〜PR〜レビュー往復〜マージ可能）を一人で回せる**

### 全フェーズ共通で重視すること
- **詰まったときに、正しい聞き方（報連相）ができる**
- **レビュー指摘に、往復で対応できる**

> 「正しい聞き方」「レビュー対応」は技術と同じくらい重視されます。**AIがどれだけ進化しても、ここは人間の仕事として残ります**。現場ではあなたの周りに先輩がいないことも多く、「何を・どう聞くか」「指摘にどう応えるか」が評価に直結します。

## 1. 進め方（毎日・毎週のリズム）

この研修は **自習（じしゅう）が中心** です。講師が横についているわけではありません。次のリズムで進めてください。

### 毎日のリズム
1. その日の課題（`exercises/`の中のファイル）を開く。
2. **まず自分で解く**。模範解答（`solutions/`）は先に見ない。
3. 各課題の「検証方法」に書かれたコマンドを叩いて、**自分で正解か判定する**。
4. 詰まったら「つまずきポイントとヒント」を見る（答えそのものではなくヒント）。
5. それでもダメなら、後述の **質問テンプレート** に沿って質問する。
6. 1日の終わりに **日報** を書く。

### 毎週のリズム
- 平日: 各自で課題を進める（自習）。
- 週1回・数時間: **対面研修**。ここは講義ではなく、「あなたが書いたコードのレビュー」「詰まりの解消」「次週課題の説明」に使います。だから、対面までに**自分のコードと質問を準備しておく**ことが大切です。

### 各週のフォルダの進む順番
```
━━ 第1フェーズ（週0〜2）: 基礎を自分の手で固める ━━
week0-setup  →  week00-it-basics  →  week1-java-basics  →  week2-sql-db
（環境構築）    （IT基礎の座学）     （Java基礎）          （SQL/DB）

━━ 第2フェーズ（週3〜12）: AIを正しく使って現場に立つ ━━
week3-springboot  →  week4-test-and-final  →  week5-ai-collab  →  week6-code-reading
（API・AI解禁）       （テスト＋総合課題）      （AI協働の作法）    （既存コード読解）
  →  week7-safe-modification  →  week8-test-quality-ai  →  week9-11-project  →  week12-final
     （安全な改修）             （テストと品質をAIで）     （実戦プロジェクト）   （総仕上げ）
```
必ずこの順で進めてください。前の週ができていないと次の週でつまずきます。
**特に、第1フェーズの修了が第2フェーズに進む前提**です。基礎が固まらないままAIを解禁すると、「中身が分からないまま貼るだけの人」になり、かえって伸びません。

**いちばん最初は [week0-setup/SETUP.md](week0-setup/SETUP.md)（環境構築）から**です。
この研修ドキュメントは、IDE **Cursor** の **MPE拡張（Markdown Preview Enhanced）** できれいに表示しながら読む想定です。そのためにまず環境（WSL2・Cursor・Git）を整え、このリポジトリを `git clone` して手元に持ってくる必要があります。環境構築の手順書（SETUP.md）の最後に、cloneとMPE表示の手順があります。

環境が整ってドキュメントをCursorで読める状態になったら、次に [week00-it-basics/README.md](week00-it-basics/README.md)（IT基礎の座学）で、環境構築中に出てきた「ターミナル」「コマンド」「サーバー」などの言葉の**意味と仕組みを腰を据えて理解**します。
全体の到達目標とスケジュールは [curriculum.md](curriculum.md) を見てください。

## 2. AI利用ルール（最重要・必ず守る）

AIエージェント（ChatGPT、Claude、Gemini など）や、この研修で使うAI内蔵IDE **Cursor** の使い方には**フェーズごとのルール**があります。各週の課題の冒頭にも、その週の利用区分を1行で示しています。

> 用語メモ: **AIエージェント** … 質問するとコードや文章を作ってくれるAIツールのこと。無料で始められるものを使ってください。
> 用語メモ: **Cursor** … この研修で使う、AI機能を内蔵したIDE（エディタ）。VS Code をベースにしていて普通のエディタとしても使えますが、AI機能には下記のとおり3つの使い方があり、フェーズで許可範囲が変わります。

### AIの「3つの使い方」を区別する（Cursorなどで共通）
AIツールには、ざっくり次の3つの使い方があります。**この区別がルールの核心**です（カッコ内はCursorでの呼び名）。

| 使い方 | 何をするか | たとえ |
|---|---|---|
| **エージェント（Cursorの Composer / Agent）** | AIが自分でコードを書き、ファイルを作る・直す | 代わりに宿題をやってもらう |
| **コード補完（Cursorの Tab補完）** | タイピング中に続きを勝手に提案してくる | 答えがチラ見えする |
| **質問（Cursorの Chat）** | 質問に**言葉で答える**だけ。コードは勝手に変えない | 家庭教師に質問する |

### 第1フェーズ（週0〜週2・基礎フェーズ）: AIに「コードを書かせる」のは禁止 🚫
許可されるのは **質問（Ask）モード = 家庭教師としての利用** だけです。

- ✅ **質問（Ask）モードは使ってよい**: エラーメッセージの意味を聞く／自分が書いたコードのレビューを依頼する／概念を説明させる（「配列とは何か、初心者向けに説明して」）
- 🚫 **エージェント（Agent）モードは使わない**: 「この課題のコードを書いて」は禁止
- 🚫 **コード補完はOFFにする**: 自分で考える前に答えが出てしまい、基礎力が付かないため。CursorのTab補完をオフにしてください（やり方は[week0-setup/SETUP.md](week0-setup/SETUP.md)のステップ3参照）

**なぜここまでするのか**: 最初の段階で自分の手でコードを書けるようにならないと、現場で通用しません。AIが出したコードを貼り付けるだけでは、エラーが出たときに直せず、設計の意図も読めません。最初に「自分で書く筋肉」をつけることが、後でAIを使いこなす土台になります。**質問モードだけ許すのは、つまずいたときに一人で前に進む力（自走力）も同時に育てたいから**です。

> この研修が3ヶ月かけて目指すのは「AIを使える人」です。にもかかわらず最初の数週間でAIを禁じるのは、矛盾ではありません。**AIが出したものを評価・修正できるのは、自分が書ける人だけ**だからです。包丁の使い方を知らない人に電動の調理器具を渡しても、料理は上達しません。だから「言語とSQLの素振り」が終わる週2までは手書きで固め、週3からAIを解禁します。順番が命です。

### 第2フェーズ（週3〜週12・AI協働フェーズ）: AIにコードを書かせてよい ✅
エージェントモードも補完もONにして構いません。ただし、ここからが本番です。**「貼るだけ」では絶対にダメ**で、次の3つが常に条件になります。

1. ✅ **生成物を読む**: AIが書いたコードを、行単位で目を通す。
2. ✅ **検証する**: 本当に動くか・正しいか・既存を壊さないかを、テストと自分の目で確かめる（[curriculum.md](curriculum.md) の「AI生成コードのレビュー観点」を毎回使う）。
3. ✅ **説明できる**: なぜこの実装なのかを、自分の言葉で言える。

各課題に「検証ノート／生成コードの説明欄」を設けています。必ず埋めてください。**説明できないコードは、たとえ動いていても提出してはいけません**。

これは現場での **AI活用の本番練習** です。AIを使ってよい現場ほど、「中身を理解せずに貼るだけ」の人は信用されません。逆に「AIを使いこなしつつ、出力を見極められる人」は強く重宝されます。この研修の3ヶ月は、**その見極めの力を体に覚えさせる**ためにあります。

## 3. 質問テンプレート（詰まったときの聞き方）

質問は **報連相（ほうれんそう = 報告・連絡・相談）の練習** です。客先では、あなたの聞き方そのものが評価されます。次のルールを守ってください。

### ルール
1. **15〜30分は自分で調べる**。エラー文をそのままネット検索する、ヒントを見る、など。
2. それでも解決しないなら、**下のテンプレートを埋めてから**質問する。

### 質問テンプレート（コピーして使う）
```
【やろうとしたこと】
（例: 週1の課題3で、入力された数字を合計するプログラムを作ろうとした）

【やったこと】
（例: for文で配列を回して合計する、というコードを書いた。コードを貼る↓）
（ここに自分のコードを貼る）

【どうなったか / 期待との違い】
（例: 合計が0になってしまう。期待は15）

【エラー全文】
（エラーが出たなら、省略せず全部貼る。出ていないなら「エラーなし」と書く）

【自分で調べたこと・試したこと】
（例: 「java for 合計」で検索して◯◯を試したが変わらなかった）
```

> ❌ ダメな質問の例:「動きません。どうすればいいですか？」
> → 何を・どうして・どうなったかが分からないので、答える側も困ります。これは現場でも同じです。

## 4. 日報フォーマット（毎日終わりに書く）

毎日、次の雛形を埋めて提出してください（提出先は講師の指示に従う）。短くてよいので**毎日**書くことが大事です。

```
■ 日報 yyyy/mm/dd（氏名）

【今日やったこと】
- （例: 週1 課題1〜3を完了。課題3はテストが全部通った）

【詰まったこと・分からなかったこと】
- （例: コレクションのMapの使い方が曖昧。getとputの違いがまだ怪しい）

【明日やること】
- （例: 週1 課題4から再開。Mapをもう一度復習する）

【今日の所感（任意）】
- （例: for文には慣れてきた）
```

## 5. フォルダ構成（どこに何があるか）

```
training/
  README.md            ← いま読んでいるこのファイル
  curriculum.md        ← 3ヶ月の全体カリキュラムと到達目標
  for-instructors.md   ← 講師向け（新人は読まなくてよい）

  ── 第1フェーズ（週0〜2）: 基礎を自分の手で固める ──
  week0-setup/         ← 【最初】環境構築の手順書（＋clone・MPE表示の手順）
  week00-it-basics/    ← IT基礎の座学（言葉と仕組み）
  week1-java-basics/   ← Java基礎（自習で小課題を多くこなす）
  week2-sql-db/        ← SQLとDBアクセス

  ── 第2フェーズ（週3〜12）: AIを正しく使って現場に立つ ──
  week3-springboot/    ← Spring BootでREST API（雛形あり・ここからAI解禁）
  week4-test-and-final/← 単体テスト＋総合課題＋コードリーディング
  week5-ai-collab/        ← AI協働の作法（任せる/任せないの線引き・検証）
  week6-code-reading/     ← 既存コード読解（AIを相棒に中規模リポを読む）
  week7-safe-modification/← 安全な改修（壊さずに直す・影響範囲）
  week8-test-quality-ai/  ← テストと品質をAIで担保する
  week9-11-project/       ← 実戦プロジェクト（既存リポ参画・PR・レビュー往復）
  week12-final/           ← 総仕上げ・最終評価
```

> 第2フェーズ（week5以降）のフォルダは、第1フェーズを進めながら順次整備していきます。各週に着手する前に、その週のREADMEが用意されているか講師に確認してください。

各週のフォルダには `README.md`（その週の狙いと課題一覧）、`exercises/`（課題）、`solutions/`（模範解答）が入っています。
