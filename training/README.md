# SES新人研修（Javaバックエンド）— はじめに必ず読むこと

このフォルダは、**IT未経験で入社した新人**が、1ヶ月（4週間）でJavaバックエンドの「現場で最初に任される実装工程」をこなせるようになるための研修一式です。

> 用語メモ: **SES（エスイーエス）** … お客様の会社（客先）に常駐して、その会社のシステム開発を手伝う働き方のこと。「常駐（じょうちゅう）」とは、自社ではなくお客様のオフィスに出向いて働くことです。

---

## 0. この研修のゴール（何ができるようになるか）

1ヶ月後、あなたが次の状態になっていることを目指します。

- **詳細設計（しょうさいせっけい）を読んで、その通りにJavaで実装できる**
  - 「詳細設計」とは、プログラムが何をどう動くべきかを文章や表で細かく書いた指示書のことです。現場では、これを「設計する側」ではなく「読んで作る側」を最初に任されます。
- **テーブルとほしいデータを示されたら、SQLを書いてJavaから実行できる**
- **詳細設計を渡されたら、REST APIを1本実装できる**
- **自分が書いたコードに単体テスト（たんたいテスト）を書ける**
- **他人が書いた既存コードを読んで、小さな改修ができる**
- **詰まったときに、正しい聞き方（報連相）ができる**

> このうち最後の「正しい聞き方」は技術と同じくらい重視されます。現場ではあなたの周りに先輩がいないことも多く、「何を・どう聞くか」が評価に直結します。

## 1. 進め方（毎日・毎週のリズム）

この研修は **自習（じしゅう）が中心** です。講師が横についているわけではありません。次のリズムで進めてください。

### 毎日のリズム
1. その日の課題（`exercises/`の中のファイル）を開く。
2. **まず自分で解く**。模範解答（`solutions/`）は先に見ない。
3. 各課題の「検証方法」に書かれたコマンドを叩いて、**自分で正解か判定する**。
4. 詰まったら「つまずきポイントとヒント」を見る（答えそのものではなくヒント）。
5. それでもダメなら、後述の **質問テンプレート** に沿って質問する。
6. 1日の終わりに **日報** を書く。

### 毎週のリズム
- 平日: 各自で課題を進める（自習）。
- 週1回・数時間: **対面研修**。ここは講義ではなく、「あなたが書いたコードのレビュー」「詰まりの解消」「次週課題の説明」に使います。だから、対面までに**自分のコードと質問を準備しておく**ことが大切です。

### 各週のフォルダの進む順番
```
week00-it-basics  →  week0-setup  →  week1-java-basics  →  week2-sql-db  →  week3-springboot  →  week4-test-and-final
（IT基礎の座学）   （環境構築）
```
必ずこの順で進めてください。前の週ができていないと次の週でつまずきます。
**いちばん最初は [week00-it-basics/README.md](week00-it-basics/README.md)（IT基礎の座学）から**です。ここで「ターミナル」「サーバー」などの言葉の意味を先に押さえてから、環境構築に進みます。
全体の到達目標とスケジュールは [curriculum.md](curriculum.md) を見てください。

## 2. AI利用ルール（最重要・必ず守る）

AIエージェント（ChatGPT、Claude、Gemini など）や、この研修で使うAI内蔵IDE **Antigravity** の使い方には**フェーズごとのルール**があります。各週の課題の冒頭にも、その週の利用区分を1行で示しています。

> 用語メモ: **AIエージェント** … 質問するとコードや文章を作ってくれるAIツールのこと。無料で始められるものを使ってください。
> 用語メモ: **Antigravity** … この研修で使う、AI機能を内蔵したIDE（エディタ）。普通のエディタとしても使えますが、AI機能には下記のとおり3つの使い方があり、フェーズで許可範囲が変わります。

### AIの「3つの使い方」を区別する（Antigravityなどで共通）
AIツールには、ざっくり次の3つの使い方があります。**この区別がルールの核心**です。

| 使い方 | 何をするか | たとえ |
|---|---|---|
| **エージェント（Agent）** | AIが自分でコードを書き、ファイルを作る・直す | 代わりに宿題をやってもらう |
| **コード補完** | タイピング中に続きを勝手に提案してくる | 答えがチラ見えする |
| **質問（Ask / チャット）** | 質問に**言葉で答える**だけ。コードは勝手に変えない | 家庭教師に質問する |

### 週1〜2（基礎フェーズ）: AIに「コードを書かせる」のは禁止 🚫
許可されるのは **質問（Ask）モード = 家庭教師としての利用** だけです。

- ✅ **質問（Ask）モードは使ってよい**: エラーメッセージの意味を聞く／自分が書いたコードのレビューを依頼する／概念を説明させる（「配列とは何か、初心者向けに説明して」）
- 🚫 **エージェント（Agent）モードは使わない**: 「この課題のコードを書いて」は禁止
- 🚫 **コード補完はOFFにする**: 自分で考える前に答えが出てしまい、基礎力が付かないため。Antigravityの設定でオフにしてください（やり方は[week0-setup/SETUP.md](week0-setup/SETUP.md)のステップ3参照）

**なぜここまでするのか**: 最初の段階で自分の手でコードを書けるようにならないと、現場で通用しません。AIが出したコードを貼り付けるだけでは、エラーが出たときに直せず、設計の意図も読めません。最初に「自分で書く筋肉」をつけることが、後でAIを使いこなす土台になります。**質問モードだけ許すのは、つまずいたときに一人で前に進む力（自走力）も同時に育てたいから**です。

### 週3〜4（応用フェーズ）: AIにコードを書かせてよい ✅
エージェントモードも補完もONにして構いません。ただし条件があります。
- ✅ 生成されたコードを **自分の言葉で説明できること**
- ✅ **なぜその実装なのか** を言えること
- 課題の中に「生成コードの説明を書く欄」を設けています。必ず埋めてください。

これは現場での **AI活用の練習** も兼ねています。現場ではAIを使ってよい場面が増えていますが、「中身を理解せずに貼るだけ」の人は信用されません。

## 3. 質問テンプレート（詰まったときの聞き方）

質問は **報連相（ほうれんそう = 報告・連絡・相談）の練習** です。客先では、あなたの聞き方そのものが評価されます。次のルールを守ってください。

### ルール
1. **15〜30分は自分で調べる**。エラー文をそのままネット検索する、ヒントを見る、など。
2. それでも解決しないなら、**下のテンプレートを埋めてから**質問する。

### 質問テンプレート（コピーして使う）
```
【やろうとしたこと】
（例: 週1の課題3で、入力された数字を合計するプログラムを作ろうとした）

【やったこと】
（例: for文で配列を回して合計する、というコードを書いた。コードを貼る↓）
（ここに自分のコードを貼る）

【どうなったか / 期待との違い】
（例: 合計が0になってしまう。期待は15）

【エラー全文】
（エラーが出たなら、省略せず全部貼る。出ていないなら「エラーなし」と書く）

【自分で調べたこと・試したこと】
（例: 「java for 合計」で検索して◯◯を試したが変わらなかった）
```

> ❌ ダメな質問の例:「動きません。どうすればいいですか？」
> → 何を・どうして・どうなったかが分からないので、答える側も困ります。これは現場でも同じです。

## 4. 日報フォーマット（毎日終わりに書く）

毎日、次の雛形を埋めて提出してください（提出先は講師の指示に従う）。短くてよいので**毎日**書くことが大事です。

```
■ 日報 yyyy/mm/dd（氏名）

【今日やったこと】
- （例: 週1 課題1〜3を完了。課題3はテストが全部通った）

【詰まったこと・分からなかったこと】
- （例: コレクションのMapの使い方が曖昧。getとputの違いがまだ怪しい）

【明日やること】
- （例: 週1 課題4から再開。Mapをもう一度復習する）

【今日の所感（任意）】
- （例: for文には慣れてきた）
```

## 5. フォルダ構成（どこに何があるか）

```
training/
  README.md            ← いま読んでいるこのファイル
  curriculum.md        ← 4週間の全体カリキュラムと到達目標
  for-instructors.md   ← 講師向け（新人は読まなくてよい）
  week00-it-basics/    ← 【最初】IT基礎の座学（言葉と仕組み）
  week0-setup/         ← 環境構築の手順書
  week1-java-basics/   ← Java基礎（自習で小課題を多くこなす）
  week2-sql-db/        ← SQLとDBアクセス
  week3-springboot/    ← Spring BootでREST API（雛形プロジェクトあり）
  week4-test-and-final/← 単体テスト＋総合課題＋コードリーディング
```

各週のフォルダには `README.md`（その週の狙いと課題一覧）、`exercises/`（課題）、`solutions/`（模範解答）が入っています。
