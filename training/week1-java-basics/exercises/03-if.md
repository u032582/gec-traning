## 1. 背景・狙い（なぜ学ぶか）

プログラムは「条件によって動きを変える」ことで役に立ちます。「残高が足りなければエラー」「点数が80以上ならB判定」など、現場のロジックはほぼ条件分岐の積み重ねです。
今回は `if` / `else if` / `else` を使って、**条件によって違う結果を返す**練習をします。あわせて、比較演算子（`>` `>=` `==`）と剰余（じょうよ＝割った余り `%`）も使います。

> 用語メモ: **条件分岐（じょうけんぶんき）** … 条件が成り立つかどうかで処理を分けること。**`%`（剰余）** … 割り算の「余り」を求める演算子。`7 % 3` は `1`。**`==`** … 「等しい」を判定する（代入の `=` とは別物）。

## 2. 詳細設計（この通りに実装する）

`Ex03` クラスに次のメソッドを実装してください。

| メソッド | 引数 | 戻り値 | 仕様 |
|---|---|---|---|
| `sign` | `int n` | `String` | 正なら`"正"`、0なら`"ゼロ"`、負なら`"負"` |
| `grade` | `int score` | `String` | 90以上`"A"` / 80以上`"B"` / 70以上`"C"` / 60以上`"D"` / それ未満`"F"` |
| `isLeapYear` | `int year` | `boolean` | うるう年なら`true`（下記ルール） |
| `max3` | `int a, int b, int c` | `int` | 3つの中の最大値 |

**うるう年のルール**（この順で考える）:
1. 400で割り切れる年は、うるう年（`true`）。
2. （1でなく）100で割り切れる年は、うるう年でない（`false`）。
3. （上記以外で）4で割り切れる年は、うるう年（`true`）。
4. どれにも当てはまらなければ、うるう年でない（`false`）。

### 自己採点用テストランナー（`Ex03Test.java`）

```java
public class Ex03Test {

    static int fail = 0;
    static int total = 0;

    static void check(String caseName, Object expected, Object actual) {
        total++;
        if (expected == null ? actual == null : expected.equals(actual)) {
            System.out.println("[PASS] " + caseName);
        } else {
            fail++;
            System.out.println("[FAIL] " + caseName + ": 期待=" + expected + ", 実際=" + actual);
        }
    }

    public static void main(String[] args) {
        Ex03 ex = new Ex03();

        check("sign+", "正", ex.sign(5));
        check("sign0", "ゼロ", ex.sign(0));
        check("sign-", "負", ex.sign(-3));

        check("gradeA", "A", ex.grade(95));
        check("gradeB", "B", ex.grade(80));
        check("gradeC", "C", ex.grade(75));
        check("gradeD", "D", ex.grade(60));
        check("gradeF", "F", ex.grade(40));

        check("leap2000", true, ex.isLeapYear(2000));
        check("leap1900", false, ex.isLeapYear(1900));
        check("leap2024", true, ex.isLeapYear(2024));
        check("leap2023", false, ex.isLeapYear(2023));

        check("max3a", 9, ex.max3(9, 4, 2));
        check("max3b", 8, ex.max3(3, 8, 5));
        check("max3c", 7, ex.max3(1, 6, 7));

        System.out.println("--------");
        if (fail == 0) {
            System.out.println("ALL PASS ✅");
        } else {
            System.out.println(fail + " 件失敗（テスト合計 " + total + " 件）");
        }
    }
}
```

## 5. つまずきポイントとヒント

<details>
<summary>grade の判定がうまくいかない（80なのにCになる等）</summary>

`if (score >= 90)` のように**大きいほうから順に**判定し、`else if` でつなぐのがコツです。`if` を独立して並べると、複数の条件に当てはまってしまいます。
</details>

<details>
<summary>うるう年の順番が大事な理由</summary>

2000年は「400で割り切れる」「100で割り切れる」「4で割り切れる」すべてに当てはまります。先に400のチェックをしないと、100のルールで `false` にしてしまいます。**400 → 100 → 4 の順**で書いてください。
</details>

<details>
<summary>max3 の考え方</summary>

まず「仮の最大値」を `a` にしておき、`b` がそれより大きければ更新、`c` も同様に比べる、という流れがシンプルです。
</details>
