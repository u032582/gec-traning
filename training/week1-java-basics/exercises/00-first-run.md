# 課題00: 最初の1本 — 自分の言葉を画面に出す

**所要時間の目安**: 20分
**AI利用区分**: 🚫 コードを書かせるのは禁止（家庭教師用途のみ）

> この課題は、週0の `Hello.java` から週1の本番課題へつなぐ**最初の成功体験**です。テストの PASS/FAIL より先に、「自分が書いた文字が画面に出た！」という実感を大事にします。

---

## 1. 背景・狙い（なぜ学ぶか）

週0の環境構築で `Hello.java` を動かしたとき、画面に文字が出ました。あれは**あなたが書いた（またはコピーした）プログラムが、コンピュータに実行された**証拠です。

この感覚をもう一度、**自分の名前入り**で体験します。正解判定は「期待どおりの文字が出たか」を**自分の目で見る**だけでOKです。

> 用語メモ: **`main` メソッド** … プログラムが「ここから始まる！」と決められた入口。`java クラス名` で実行すると、ここが最初に呼ばれます。

---

## 2. 詳細設計（この通りに実装する）

`FirstApp.java` というファイルを作り、次の仕様どおりに書いてください。

| 項目 | 仕様 |
|---|---|
| クラス名 | `FirstApp`（ファイル名も `FirstApp.java`） |
| `main` メソッド | 必ず1つ用意する |
| 画面出力 | 次の2行を**この順番**で表示する |

```
こんにちは、（あなたの名前）さん！
Javaプログラムを動かせました 🎉
```

（あなたの名前）の部分は、本名でもニックネームでも構いません。例:

```
こんにちは、田中さん！
Javaプログラムを動かせました 🎉
```

### ひな型（`FirstApp.java`）

```java
public class FirstApp {
    public static void main(String[] args) {
        // TODO: ここに2行の System.out.println を書く
    }
}
```

> 用語メモ: **`System.out.println(...)`** … 画面（ターミナル）に1行出力して、改行する命令。

---

## 3. 動かしてみる（完成条件）

1. 作業フォルダを用意する（例: `~/work/week1`）。
2. `FirstApp.java` を保存する。
3. 次のコマンドを実行する。

```bash
javac FirstApp.java && java FirstApp
```

4. **画面に自分の名前入りの2行が出たら合格**です。

### 期待される出力の例

```
こんにちは、田中さん！
Javaプログラムを動かせました 🎉
```

> 💡 ここで大事なのは `[PASS]` や `ALL PASS ✅` ではなく、**自分の作ったプログラムが動いた実感**です。この感覚を覚えておいてください。課題01以降も、テストの前に必ず「動かしてみる」ステップがあります。

---

## 4. つまずきポイントとヒント

<details>
<summary>「class FirstApp is public, should be declared in a file named FirstApp.java」と出る</summary>

`public class` の名前とファイル名は一致させる必要があります。クラス名を `FirstApp` に、ファイル名を `FirstApp.java` にしてください。
</details>

<details>
<summary>「command not found: javac」と出る</summary>

JDKが入っていないか、WSL2のターミナルで実行していない可能性があります。[week0-setup/SETUP.md](../../week0-setup/SETUP.md) の環境構築を見直してください。
</details>

<details>
<summary>コンパイルは通るが何も出ない</summary>

`main` メソッドの中に `System.out.println(...);` を書きましたか？ セミコロン `;` を忘れていないか確認してください。
</details>
