全体の進め方・AI利用ルール・質問テンプレート・日報フォーマットは、先に [研修トップのREADME](../README.md) を必ず読んでください。

## 1. この週の狙い（到達目標）

> **詳細設計（しょうさいせっけい）レベルの仕様を渡されたら、Javaで書ける。**

「詳細設計レベルの仕様」とは、「このメソッドは、こういう引数を受け取って、こういう値を返す」というところまで細かく決められた指示書のことです。現場であなたが最初に任されるのは、この「決められた通りに作る」工程です。だからこの週の課題は、すべて **メソッド名・引数の型・戻り値の型まで指定** してあります。あなたの仕事は「設計どおりに中身を実装する」ことです。

この週で学ぶ範囲:

- 変数（へんすう）と出力
- 四則演算（しそくえんざん）と型変換（かたへんかん）
- 制御構文（せいぎょこうぶん）: `if` / `for` / `while`
- 配列（はいれつ）
- コレクション: `List`（リスト） / `Map`（マップ） / `Set`（セット）
- 例外処理（れいがいしょり）
- クラス設計（クラスせっけい）
- Stream API（ストリームエーピーアイ）

> 用語メモ: **メソッド** … 処理のまとまりに名前を付けたもの。「関数（かんすう）」とほぼ同じ意味。**引数（ひきすう）** … メソッドに渡す入力。**戻り値（もどりち）** … メソッドが返す出力。

## 4. 進め方（1課題のサイクル）

各課題は、次のサイクルで進めます。**テストの前に必ず「動かしてみる」** ステップを挟みます。

1. `exercises/NN-xxx.md` を開いて、**仕様（詳細設計）を読む**。
2. 指定されたファイル名・クラス名・メソッド名で、**自分でコードを書く**（`solutions/` は先に見ない）。
3. **動かしてみる用ランナー**（`ExNNPlay.java`）を実行して、**画面に結果が出ることを確認する**。
4. 自己採点用テストランナー（`ExNNTest.java`）を実行して、**自分で正解か判定する**。
5. `ALL PASS ✅` が出れば完成。出なければ、失敗したケースの表示を読んで自分で直す。
6. どうしても詰まったら、課題の「つまずきポイントとヒント」（折りたたみ）を開く。
7. それでもダメなら、模範解答 `solutions/ExNN.java` を見て、**自分のコードと比べる**。

> **なぜ2段階？** `[PASS]` / `[FAIL]` だけだと「動いた実感」が薄く、続ける力（自己効力感）が育ちにくいからです。まず画面に**自分の作った結果**を見てから、テストで正確さを確認してください。

## 5. 「動かしてみる」と「正解判定」の2段階

この週には、**2種類のランナー** があります。

| ファイル | 役割 | いつ使うか |
|---|---|---|
| `ExNNPlay.java` | あなたのコードを呼び出し、**結果を人間が読める形で画面に出す** | 実装の直後（**最初**） |
| `ExNNTest.java` | 期待値と比較して **PASS/FAIL を判定** | Play で動いたあと（**2番目**） |

課題00（最初の1本）だけは Play 相当の `main` を自分で書き、テストランナーはありません。

### 動かしてみる（例: 課題01）

```bash
javac Ex01.java Ex01Play.java && java Ex01Play
```

画面に `こんにちは、田中さん！` のような**自分の作った文章**が出れば、まず成功です。

### 正解判定（例: 課題01）

```bash
javac Ex01.java Ex01Test.java && java Ex01Test
```

`ALL PASS ✅` が出れば、その課題は完了です。

## 6. 自己採点の仕組み（テスト実行方法）

この週のテストは、**外部のテスト道具（JUnitなど）を一切使いません**。`javac` と `java` だけで動きます。新人が一人で正解判定できることを最優先にしています。

各課題には、次の2つのJavaファイルが関係します。

- `ExNN.java` … **あなたが実装するファイル**（最初は中身がない／一部だけ用意されている）。
- `ExNNTest.java` … **自己採点用テストランナー**。`main` メソッドの中で、あなたの実装を何度も呼び出して、結果が期待どおりかを `assert`（チェック）します。

### 実行手順（全課題共通）

テストランナー（`ExNNTest.java`）は、各課題ファイルの中に全文を載せてあります。それを `solutions/` と同じ場所、あるいは自分の作業フォルダにコピーして、次のように実行します（例は課題06の場合）。

```bash
# 1. 自分の作業フォルダに移動（例）
cd ~/work/week1

# 2. 自分の実装ファイルとテストランナーを同じフォルダに置く
#    Ex06.java（自分の実装） と Ex06Test.java（課題からコピー）

# 3. 両方まとめてコンパイル
javac Ex06.java Ex06Test.java

# 4. テストランナーを実行
java Ex06Test
```

### 出力の読み方

- 全部正解なら、最後に次のように出ます。

  ```
  ALL PASS ✅
  ```

- どこかが間違っていると、次のように「どのケースが・何を期待して・実際どうだったか」が出ます。

  ```
  [FAIL] case3: 期待=15, 実際=0
  1 件失敗（テスト合計 5 件）
  ```

  この表示を読めば、自分のどこが間違っているか見当がつきます。これも現場で必要な「原因の切り分け」の練習です。

> メモ: このテストランナーは、Javaの `assert` キーワードではなく、自前のチェックメソッドを使っています。なので `java -ea`（assert有効化オプション）を付ける必要はありません。普通に `java ExNNTest` で動きます。
