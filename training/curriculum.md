# 4週間カリキュラム — 全体像と到達目標

このファイルは、研修全体の地図です。「いま自分がどこにいて、次に何ができるようになるのか」を確認するために使ってください。
進め方・AI利用ルール・質問の仕方は [README.md](README.md) を見てください。

---

## 全体の流れ（1ヶ月）

```
週0前半 IT基礎(座学)  : ITの「言葉」と「仕組み」をイメージできるようになる
週0後半 環境構築      : 道具をそろえる（PCで開発できる状態にする）
週1     Java基礎     : 自分の手でJavaが書けるようになる
週2     SQLとDBアクセス: データベースからデータを出し入れできるようになる
週3     Spring Boot  : Webの裏側（API）を1本作れるようになる
週4     テスト＋総合課題: テストを書き、実装〜提出まで一人で回せるようになる
```

各週は「前の週の上に積み上がる」設計です。順番を飛ばさないでください。
**いちばん最初は週0前半の「IT基礎（座学）」**です。ここを飛ばすと、後で出てくる言葉が分からず必ず詰まります。

## 各週の到達目標（この状態になればクリア）

### 週0前半: IT基礎（座学）
- **学ぶこと**: ハード/ソフト、OS（とLinux/WSL2）、ファイル/フォルダ/パス、ターミナルとコマンド、プログラムが動く仕組み（コンパイル/実行/エラー）、ネットワーク/サーバー/クライアント、フロント/バックエンド、データベースの位置づけ。
- **到達目標**: ITの基本の言葉を**自分の言葉で説明でき**、開発の全体像をイメージできる。
- **完了の判定**: [week00-it-basics/README.md](week00-it-basics/README.md) の各章の理解チェックとハンズオン小課題をクリアできる。
- **AI利用区分**: 質問（Ask）モードのみ（言葉の意味を聞く等の家庭教師用途）。
- **狙い**: IT未経験者がいきなり環境構築で詰まらないよう、先に「言葉と仕組み」を入れて土台を作る。

### 週0後半: 環境構築
- **到達目標**: WSL2(Linux)上でJavaプログラムをビルド・実行でき、Gitで変更を記録でき、PostgreSQLに接続できる。
- **完了の判定**: SETUP.mdの最後にある「動作確認チェックリスト」が全部✅になる。
- **AI利用区分**: 家庭教師用途のみ（エラーの意味を聞く等）。

> 用語メモ: **ビルド** … 人間が書いたコードを、コンピュータが実行できる形に変換すること。**PostgreSQL（ポストグレスキューエル）** … データを保存する「データベース」の一種。

### 週1: Java基礎を実装で固める
- **学ぶこと**: 変数 / 制御構文（if・for・while）/ 配列 / コレクション（List・Map・Set）/ 例外処理 / クラス設計 / Stream API。
- **到達目標**: 「詳細設計レベルの仕様を渡されたら、Javaで書ける」。
- **やり方**: 小さな課題を**数多く**こなす。各課題に自己採点用のテストか期待出力が付いている。
- **完了の判定**: 各課題のテストが通る／期待出力と一致する。
- **AI利用区分**: 🚫 コードを書かせるのは禁止。家庭教師用途のみ。

### 週2: SQLとDBアクセス
- **学ぶこと**: SELECT / JOIN / 集計（GROUP BY等）/ INSERT / UPDATE / DELETE、正規化（せいきか）とトランザクションの基礎、MyBatisでJavaからDBを叩く。
- **到達目標**: 「テーブルと取得したいデータを示されたら、SQLを書け、Javaから実行できる」。
- **完了の判定**: SQLの実行結果が期待と一致する／自己採点テストが通る。
- **AI利用区分**: 🚫 コードを書かせるのは禁止。家庭教師用途のみ。

> 用語メモ: **JOIN（ジョイン）** … 複数の表をつなげて1つの結果にすること。**トランザクション** … 「一連の処理をまとめて、全部成功か全部取り消すか」を保証する仕組み。**MyBatis（マイバティス）** … JavaとSQLをつなぐ道具。この研修ではJPAではなくMyBatisに絞ります。

### 週3: Spring BootでREST API
- **学ぶこと**: 3層構造（Controller / Service / Repository）、DI（ディーアイ）、リクエスト/レスポンス、Bean Validation（入力チェック）、例外ハンドリング、MyBatis連携。
- **到達目標**: 「詳細設計を渡されたら、APIを1本実装できる」。
- **やり方**: ビルドが通る**雛形プロジェクト**を渡すので、指定のエンドポイントを1本ずつ追加していく。
- **完了の判定**: 指定のリクエストを送ると、設計どおりのレスポンスが返る。
- **AI利用区分**: ✅ コードを書かせてよい。ただし生成コードを自分の言葉で説明できること。

> 用語メモ: **REST API（レストエーピーアイ）** … プログラム同士がインターネット越しにデータをやり取りする窓口。**3層構造** … 役割を「窓口（Controller）」「判断・処理（Service）」「DB出し入れ（Repository）」の3つに分ける作り方。**DI** … 必要な部品を外から差し込んでもらう仕組み（依存性の注入）。**エンドポイント** … APIの個々の窓口（URL）。

### 週4: 単体テスト＋総合課題＋コードリーディング
- **学ぶこと**: JUnit5 / Mockito での単体テスト。小さなCRUD APIを詳細設計付きで実装し、テストを書き、Gitでプルリク（PR）提出まで。さらに「他人が書いた既存コードを読んで小改修する」課題を1つ。
- **到達目標**: 「現場の実装工程を一通り自走できる」。
- **完了の判定**: 総合課題のAPIが設計どおり動き、テストが通り、PRが出せている。改修課題は改修後にテストが通る。
- **AI利用区分**: ✅ コードを書かせてよい。ただし説明できること。

> 用語メモ: **単体テスト** … 1つの部品（関数やクラス）が正しく動くかを、自動で確認するコード。**JUnit5 / Mockito** … Javaで単体テストを書くための道具。**CRUD（クラッド）** … Create（作成）/ Read（読む）/ Update（更新）/ Delete（削除）の4操作。**プルリク（PR）** … 「このコードを取り込んでください」と提出する仕組み。
