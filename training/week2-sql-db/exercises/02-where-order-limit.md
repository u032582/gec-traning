# 課題02: 絞り込み・並べ替え・件数制限 — WHERE / ORDER BY / LIMIT

- **所要時間の目安**: 40分
- **AI利用区分**: 🚫 AIにSQLを書かせるのは禁止。家庭教師用途のみ（エラーの意味/レビュー/概念説明）。

> 用語メモ:
> **WHERE（ウェア）** … 「条件に合う行だけ」に絞り込む。
> **ORDER BY（オーダーバイ）** … 結果を指定した順に並べ替える。
> **LIMIT（リミット）** … 先頭から指定した件数だけに制限する。

---

## 背景・狙い（現場でどう使うか）

全件をそのまま見ることは現場ではほとんどありません。「3000円以上の商品だけ」「飲料カテゴリだけ」「高い順トップ3」のように、**条件で絞り、順に並べ、必要な分だけ**取り出すのが普通です。この3つの組み合わせはSELECTの実戦の中心です。

引き続き `products`（商品）テーブルだけを使います。

---

## 仕様（やること）

次の4問を、それぞれ1つのSELECT文で解いてください。

- **問1（WHERE）**: 単価（`price`）が **1000円以上** の商品の **全列** を取り出す。
- **問2（WHERE）**: カテゴリ番号（`category_id`）が **2（飲料）** の商品の、**商品名（name）と単価（price）** を取り出す。
- **問3（ORDER BY）**: **全商品** の **商品名（name）と単価（price）** を、**単価の高い順（降順）** に並べて取り出す。
- **問4（ORDER BY + LIMIT）**: 全商品のうち、**単価が安い順（昇順）に上位3件** の **商品名（name）と単価（price）** を取り出す。

> 用語メモ: **降順（こうじゅん）= DESC** … 大きい順。**昇順（しょうじゅん）= ASC** … 小さい順。`ORDER BY price DESC` のように書きます。ASC は省略しても昇順になります。

---

## 完成条件（自分で正解か判定する基準）

- 問1: **5行**（1000円以上の商品）。
- 問2: **3行**（飲料カテゴリの商品）。
- 問3: **12行**。先頭が `ノートPC`、末尾が `ミネラルウォーター` になる。
- 問4: **3行**。`ミネラルウォーター(90) → バナナ(100) → 緑茶(130)` の順。

## つまずきポイントとヒント

<details>
<summary>「1000円以上」の比較記号は？</summary>

「以上」は `>=`、「より大きい」は `>`、「以下」は `<=`、「未満」は `<`、「等しい」は `=`（イコール1つ。プログラミングと違って2つではない）です。「1000円以上」なら `price >= 1000`。
</details>

<details>
<summary>WHERE と ORDER BY と LIMIT を一緒に使うときの順番</summary>

書く順番は決まっています: `SELECT 列 FROM 表 WHERE 条件 ORDER BY 並べ替え LIMIT 件数;`。この順を崩すとエラーになります。
</details>

<details>
<summary>「安い順トップ3」の作り方</summary>

「安い順」= 単価の昇順 = `ORDER BY price`（ASCは省略可）。「トップ3」= `LIMIT 3`。この2つを組み合わせます。
</details>

<details>
<summary>文字列の条件を書くとき（今回は使わないが先取り）</summary>

文字列はシングルクォートで囲みます（例: `WHERE name = 'りんご'`）。ダブルクォート `"` は列名・別名用なので、値に使うとエラーになります。よくある間違いなので覚えておくと得です。
</details>
