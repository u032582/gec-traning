# 課題01: SELECT の基礎 — 表からデータを取り出す

- **所要時間の目安**: 30分
- **AI利用区分**: 🚫 AIにSQLを書かせるのは禁止。家庭教師用途のみ（エラーの意味/レビュー/概念説明）。

> 用語メモ: **SELECT（セレクト）** … 「データを取り出して見せて」とDBに頼む命令。SQLで一番よく使います。

---

## 背景・狙い（現場でどう使うか）

現場でまず任されるのは「このテーブルの、この列を見たい」という**データの確認**です。トラブル調査でも「該当データが本当に入っているか」をSELECTで確かめます。SELECTは、DBを触る人にとっての「呼吸」のようなもの。まずはこれを体に入れます。

この課題では、`products`（商品）テーブルだけを使います。

---

## 仕様（やること）

`products` テーブルには次の列があります（`schema.sql` 参照）。

| 列名 | 意味 |
|---|---|
| `id` | 商品番号 |
| `name` | 商品名 |
| `category_id` | カテゴリ番号 |
| `price` | 単価（円） |
| `stock` | 在庫数 |

次の3問を、それぞれ1つのSELECT文で解いてください。

- **問1**: `products` の **全行・全列** を取り出す。
- **問2**: `products` から **商品名（name）と単価（price）の2列だけ** を取り出す（全行）。
- **問3**: `products` から **商品名（name）だけ** を取り出し、列の見出し（表示名）を `商品名` という日本語にする。

> 用語メモ: **列の別名（エイリアス）** … `SELECT name AS 商品名` のように `AS 別名` と書くと、結果の見出しを好きな名前に変えられます。問3で使います。

---

## 完成条件（自分で正解か判定する基準）

- 問1: 12行が返り、列は `id, name, category_id, price, stock` の5つ。
- 問2: 12行が返り、列は `name, price` の2つだけ。
- 問3: 12行が返り、見出しが `商品名` になっている。

行数（12行）は `products` の全商品数です。

### 答え合わせ用の確認SQL（件数チェック）

自分の問1のSQLを、下の `（ここに自分のSELECT文）` の部分に丸ごと貼り付けて実行すると、**件数が機械的に確認**できます。

```sql
SELECT COUNT(*) FROM ( （ここに自分のSELECT文） ) AS t;
```

例:

```sql
SELECT COUNT(*) FROM ( SELECT * FROM products ) AS t;
```

**期待される結果**: `12`

> このやり方は以降の課題でも使えます。「自分のSELECTをカッコで囲んで、その件数を数える」ことで、行数が合っているかを目視より確実にチェックできます。

## つまずきポイントとヒント

<details>
<summary>「全行・全列」ってどう書くの？</summary>

「全部の列」は `*`（アスタリスク）という記号で表します。`SELECT （列の指定） FROM （表名）;` の形を思い出してください。文の最後のセミコロン `;` を忘れると、psqlは「まだ続きがある」と思って実行してくれません。
</details>

<details>
<summary>特定の列だけ欲しいときは？</summary>

`SELECT` の後ろに、欲しい列名をカンマ `,` で区切って並べます。順番は自分で決められます。
</details>

<details>
<summary>見出しを日本語にしたい（問3）</summary>

`列名 AS 別名` の形です。別名に記号や空白を含めたいときはダブルクォートで囲みますが、`商品名` のような単純な日本語ならそのままでも通ります。
</details>

<details>
<summary>エラーが出たときの読み方</summary>

`column "xxx" does not exist`（そんな列はない）なら列名のタイプミス、`relation "xxx" does not exist`（そんな表はない）なら表名のタイプミスです。エラー文の意味が分からなければ、AIに「このPostgreSQLのエラーの意味を初心者向けに教えて」と全文を貼って聞いてください（コードは書かせない）。
</details>
