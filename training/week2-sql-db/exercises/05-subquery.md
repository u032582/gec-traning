# 課題05: サブクエリ — クエリの中のクエリ

- **所要時間の目安**: 50分
- **AI利用区分**: 🚫 AIにSQLを書かせるのは禁止。家庭教師用途のみ（エラーの意味/レビュー/概念説明）。

> 用語メモ: **サブクエリ（副問い合わせ）** … SELECT文の中に、もう1つのSELECT文を入れ子にすること。「まず内側のSELECTで“ある値”や“ある一覧”を求めておき、それを外側のSELECTの条件に使う」という2段構えの書き方です。

---

## 背景・狙い（現場でどう使うか）

「平均より高い商品」「一度でも注文された商品」「東京の顧客の注文だけ」——こうした条件は、**まず別のSELECTで基準を求めてから絞り込む**と素直に書けます。これがサブクエリです。JOINと並んで、複雑な抽出依頼に応えるための定番の道具。最初は入れ子が読みにくいですが、「内側→外側」の順で読めば理解できます。

---

## 仕様（やること）

次の4問を解いてください。

- **問1（値を返すサブクエリ）**: **顧客番号4（田中 美咲）よりも前に登録した顧客** の **氏名（name）と登録日（registered_at）** を出す。
  - 「顧客4の登録日」をサブクエリで求め、それより前（`<`）の顧客を外側で絞る。
  - ヒント: `WHERE registered_at < (SELECT registered_at FROM customers WHERE id = 4)`
  - 使う表: `customers` のみ。
- **問2（IN を使うサブクエリ）**: **一度でも注文された商品** の **商品名（name）** を出す。
  - 「注文明細に登場する商品番号の一覧」をサブクエリで求め、それに含まれる商品を外側で絞る。
  - ヒント: `WHERE id IN (SELECT product_id FROM order_items)`
  - 使う表: `products`、サブクエリで `order_items`。
- **問3（NOT IN を使うサブクエリ）**: 問2の逆。**一度も注文されたことがない商品** の **商品名（name）** を出す。
  - ヒント: `WHERE id NOT IN (SELECT product_id FROM order_items)`
- **問4（IN + 条件つきサブクエリ）**: **東京都の顧客がした注文** の **注文番号（id）** を出す。
  - 「東京都の顧客の顧客番号一覧」をサブクエリで求め、その顧客の注文を外側で絞る。
  - ヒント: `WHERE customer_id IN (SELECT id FROM customers WHERE prefecture = '東京都')`
  - 使う表: `orders`、サブクエリで `customers`。

---

## 完成条件（自分で正解か判定する基準）

- 問1: **3行**（田中より前に登録した3人）。
- 問2: **11行**（注文されたことがある商品は12品中11品）。
- 問3: **1行**（`洗剤` だけが一度も注文されていない）。
- 問4: **5行**（東京都の顧客の注文）。

### 答え合わせ用の確認SQL（件数チェック）

```sql
-- 問1は 3
SELECT COUNT(*) FROM ( （ここに問1のSELECT文） ) AS t;
-- 問2は 11
SELECT COUNT(*) FROM ( （ここに問2のSELECT文） ) AS t;
-- 問3は 1
SELECT COUNT(*) FROM ( （ここに問3のSELECT文） ) AS t;
-- 問4は 5

-- 問2と問3の件数を足すと 12（全商品数）になるはず（注文あり11 + 注文なし1）
```

## つまずきポイントとヒント

<details>
<summary>サブクエリってどう読めばいい？</summary>

**内側のSELECTを先に頭の中で実行**してください。問1なら、内側 `SELECT registered_at FROM customers WHERE id = 4` はまず `2024-02-01` という1つの値になります。外側はそれを使って `WHERE registered_at < '2024-02-01'` と同じ意味になります。
</details>

<details>
<summary>「= ではなく IN を使うのはどんなとき？」</summary>

サブクエリが**1つの値**を返すなら `=` や `<` で比べられます（問1）。**複数の値（一覧）**を返すなら `IN`（その中に含まれる）を使います（問2・問4）。「複数返るのに = で比べる」とエラーになります。
</details>

<details>
<summary>NOT IN の注意点（問3）</summary>

`NOT IN (...)` は「その一覧に含まれない」。今回の `order_items.product_id` には空（NULL）が無いので素直に動きます。一般にサブクエリ側にNULLが混ざると NOT IN は直感と違う動きをすることがある、という点だけ頭の隅に置いておくと将来役立ちます（今は気にしなくてOK）。
</details>

<details>
<summary>文字列の条件（問4の '東京都'）</summary>

文字列はシングルクォートで囲みます: `prefecture = '東京都'`。ダブルクォートにするとエラーです。
</details>
