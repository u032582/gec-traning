# 課題3 模範解答: ShippingFeeCalculator の改修と「悪かった点」の解説

このフォルダは、課題3（コードリーディング+小改修）の模範解答です。
- `src/main/java/com/example/legacy/ShippingFeeCalculator.java` … 改修後コード
- `src/test/java/com/example/legacy/ShippingFeeCalculatorCharacterizationTest.java` … `legacy-code` と**同一**の振る舞い固定テスト（改修後も全部緑）

> 確認: このフォルダで `./gradlew test` を流すと、改修前と同じテストが全部通ります。
> = 見た目を直しても外から見た振る舞いを変えていない、という証拠です。

---

## 改修前コードの「悪かった点」と、その理由・直し方

### ① 重複コード（同じ集計ロジックのコピペ）
**何が悪い**: `calc` と `calcWithDiscount` の中に、「地域と重量から料金を出す」ほぼ同じコードが2回書かれていた。
**なぜ悪い**: 料金表を1つ変えるたびに2か所直す必要があり、片方を直し忘れると**バグになる**。重複は「将来の修正コスト」を倍にする。
**直し方**: 1行ぶんの料金を計算する `feeForLine(String line)` に切り出し、両メソッドから呼ぶ。`calcWithDiscount` は `calc` の結果を再利用するだけにした。これで料金ルールの修正は1か所で済む。

### ② 無意味な変数名（`a` / `tmp` / `data2`）
**何が悪い**: `a`（合計）, `tmp`（その行の料金）, `data2`（分割した列）といった名前は、何を表すか読んで分からない。
**なぜ悪い**: 読み手（未来の自分や他人）が、変数の意味を毎回コードから推理しなければならず、読むのが遅くなり誤解も生む。**命名は最良のコメント**。
**直し方**: `total` / `columns` / `regionCode` / `weight` など、中身が名前から分かる名前に変更した。

### ③ 例外の握りつぶし（空の `catch`）
**何が悪い**: `catch (Exception e) {}` で、エラーが起きても何も記録せず黙って捨てていた。さらに `Exception` という広すぎる型で、想定外の不具合まで飲み込んでしまう。
**なぜ悪い**: 壊れたデータが来ても誰も気づけない。本番でデータの取りこぼしが起きても、ログに何も残らず**原因調査が不可能**になる。これは現場で最も嫌われるアンチパターンの1つ。
**直し方**: 「壊れた行は集計に含めない」という**結果（振る舞い）は保ったまま**（=テストを赤くしない）、原因を `System.err` に出力するようにした。さらに捕まえる例外を、実際に起こりうる `NumberFormatException`（数値変換失敗）と `ArrayIndexOutOfBoundsException`（列不足）に絞った。
> 補足: 研修なので標準エラー出力にしているが、現場では Logger（SLF4J等）を使うのが定石。

### ④ マジックナンバー（`500` `2000` `20` など）
**何が悪い**: 料金や重量の区切り、割引率がコードに直接埋め込まれていて、何の数字か説明がない。
**なぜ悪い**: 「2000」が重量の境目なのか金額なのか、読まないと分からない。同じ数字が複数箇所にあると変更漏れも起きる。
**直し方**: `LIGHT_WEIGHT_LIMIT` / `MEDIUM_WEIGHT_LIMIT` のような名前付き定数にし、地域ごとの料金は `Region` enum（列挙型）の表にまとめた。割引率は `switch` の表に整理した。

### ⑤ 長い・見通しの悪いメソッド
**何が悪い**: ネストの深い if が連なり、1メソッドが何でもやっていた。
**なぜ悪い**: 1メソッドの責務が多いほど、読むのも直すのも難しくなる。
**直し方**: 「1行を料金にする」「地域を判定する」「割引率を引く」を別々のメソッド/enumに分け、各部分を短く一目で読める単位にした。

## 改修で守ったこと（重要）

- **戻り値は一切変えていない**。だから `legacy-code` と同じ characterization test が全部緑のまま。
- 「壊れた行をスキップして合算に含めない」という既存の振る舞いも保った（握りつぶしを“ログを残す”に変えただけ）。

リファクタリングは「動きを変えずに中身を良くする」作業です。**テストという安全網があるから、思い切って中身を整理できる**——この感覚を持ち帰ってください。
